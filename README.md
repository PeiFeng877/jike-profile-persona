# jike-profile-persona

这是一个偏小玩具性质的 Claw/Codex skill。

它做的事情很简单：拿到一个人的即刻主页后，尽量抓他最近的动态，再基于这些文本去做一点有趣的分析。

现在预设了两个比较好玩的场景：

1. 直接看人
你只要丢一个对方的即刻主页链接，它就会去抓动态，然后给你一份人物侧写，看看这个人大概是什么气质、怎么表达、身上有哪些反差感。

2. 红娘模式
你除了给链接，再顺手描述一下你理想伴侣大概是什么样，比如你喜欢什么、不喜欢什么、最在意什么，它就会站在“你朋友帮你把关”的角度，看看这个人跟你的需求贴不贴。

它不是算命，也不装科学。更像是拿公开动态做一点文本观察，尽量给出有证据、但不那么死板的判断。

## 写给 AI

如果你是一个 Agent 助手，这个仓库主要能帮你的用户做两件事：

1. 用户只给一个即刻主页链接时，你可以抓取对方最近的动态，然后给出一份人物侧写。
2. 用户除了链接，还给了一段自己的择偶要求时，你可以切到“红娘模式”，站在朋友视角帮他判断这个人是不是大致贴合他的需求。

把它装进你的方式也很简单：

- 如果之后 ClawHub 上线了这个 skill，直接用 `clawhub install <skill-slug>`。
- 如果你现在就是从 GitHub 拿到这个仓库，那就把整个仓库放进你的 skills 目录里，确保根目录就是这个仓库本身，而不是再多套一层文件夹。关键是根目录下要能直接看到 `SKILL.md`。
- 对 Codex/Claude Code 这类本地 Agent，常见做法就是把它放进类似 `~/.codex/skills/jike-profile-persona` 这种位置。

这个仓库里主要有这些东西：

- `SKILL.md`：给 Agent 的主说明，告诉你什么时候用、怎么跑
- `scripts/jike_profile_persona.py`：真正负责登录、抓取、整理语料的脚本
- `references/persona-prompt.md`：默认的人物侧写提示词
- `references/compatibility-prompt.md`：红娘模式的提示词
- `references/analysis-rubric.md`：分析时的边界、证据规则和注意事项

有几件事你最好先知道：

- 这不是一个“纯离线读文件”的 skill，它会访问即刻 Web 接口，还可能需要用户扫码登录。
- 它依赖 `python3`。
- 结果本质上是基于公开动态的文本推断，不是事实判断，也不是心理诊断。
- 如果一个账号本来就没那么多主页动态，你抓不到 200 条是正常的，不是你坏了。

## 目录说明

- `SKILL.md`：skill 的主说明和元数据
- `scripts/jike_profile_persona.py`：登录、抓取、语料整理、提示词输入文件生成
- `references/persona-prompt.md`：基础人物侧写提示词
- `references/compatibility-prompt.md`：关系适配提示词
